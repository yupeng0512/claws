## 核心审美原则

这些是筛选一切信息的底层价值观，优先级从高到低：

1. **深度优于广度** — 一个真正理解透彻的洞察，胜过十个浅尝辄止的信息
2. **原创优于衍生** — 第一手的创造和思考 > 对他人工作的总结和评论
3. **实践优于理论** — 有人真正用它构建了什么 > 只是概念论证
4. **反直觉优于确认** — 挑战既有认知的发现 > 强化已知结论的信息
5. **长期优于短期** — 3 年后仍然重要的 > 下周就过时的
6. **优雅优于复杂** — 用更少的东西解决更多的问题

## 筛选打分标准（详细版）

### 新颖性 (1-5)
- 5: 全新的范式/方法，从未见过
- 4: 已知方向的重大突破
- 3: 有意思的新组合或新应用
- 2: 增量改进
- 1: 旧概念换皮

### 深度 (1-5)
- 5: 有严谨的技术论证/实验数据/源码
- 4: 有实质性的架构设计或深入分析
- 3: 有一定深度但仍在表面
- 2: 偏概述/入门
- 1: 只是标题党或新闻稿

### 实用性 (1-5)
- 5: 今天就能用，直接解决现有问题
- 4: 1-2 周内可以集成
- 3: 需要 1 个月的适配
- 2: 有启发但短期用不上
- 1: 纯学术/理论

### 趋势信号 (1-5)
- 5: 被 5+ 独立来源同时提到，明显趋势
- 4: 多个早期信号指向同一方向
- 3: 有一些关注但还未形成共识
- 2: 孤立事件
- 1: 可能是昙花一现

### 品味匹配 (1-5)
- 5: 完美符合核心审美原则 + Tier 1 领域
- 4: 符合核心原则 + Tier 2 领域
- 3: 部分符合，有一些有趣的角度
- 2: 边缘相关
- 1: 不在关注范围

## 反模式（主动跳过的内容）

这些类型的内容无论分数多高都直接跳过：

- **炒作型**: "X will replace Y forever!" 没有技术论证的断言
- **列表型水文**: "Top 10 AI tools in 2026" 这种聚合内容
- **身份政治型**: 任何关于"AI 会不会取代程序员"的争论
- **Vaporware**: 只有 demo 视频没有源码/文档的项目
- **套壳项目**: 只是在别人 API 上包了一层 UI 没有创新
- **Token 驱动**: 主要靠发币/空投驱动的项目
- **二手聚合型**: 来源为 CSDN/知乎/腾讯新闻等平台且无法追溯原始来源的内容，直接降级，不进入 deep_dive（可作为线索追溯原始来源后重新评估）
- **冗余重复型**: 与同批次已收录内容高度重叠（>70% 信息重合）的条目，即使来源不同也跳过
- **幻觉项目型**: 无法通过直接访问 URL 验证真实存在的项目，直接从候选列表移除。GitHub 项目必须能访问到真实仓库（有 README、有 commit 历史），不接受"看起来合理"的项目名称。证据：Ruflo（https://github.com/ruflo）真实性未经验证，占用筛选配额但未产生价值

## 扫描质量标准

_这是对扫描过程本身的约束，不是对内容的约束。_

- **最低采集量**: 每次扫描至少采集 **15 条**候选，筛选后保留 3-8 条（目标通过率 20-30%）。**低于 15 条的扫描视为失败，必须重新执行**
- **强制覆盖源**: 每次扫描必须覆盖 GitHub Trending（≥10 条候选）+ HN Top 30（≥5 条候选），缺失任一视为不完整扫描
- **英文源覆盖**: 每次扫描至少包含 2 个英文一手源（GitHub Trending、HN、Simon Willison 等）
- **来源可追溯**: 每条入选内容必须有可验证的原始来源链接，不接受"企业架构分析"这类模糊来源标注
- **去重检查**: 同一事件/项目的多个报道只保留来源最权威、信息最完整的一条
- **真实性验证**: GitHub 项目必须能访问到真实仓库页面，不接受无法验证的项目链接
- **Analyst 去重**: 同一批次的 deep-dive 报告不得重复分析相同项目；如果两次 Analyst 运行分析了相同候选集，第二次运行应被跳过
